# Dot2Dot 邮件读取脚本 · 工作逻辑说明

## 📌 项目概述

这是一个 Python 脚本，用于从指定邮箱（当前使用 QQ 邮箱）读取未读邮件，提取邮件内容，并写入 Excel 数据库（`database1.xlsx`）的 `sheet2` 工作表中。

---

## 🔄 核心工作流程

```
启动脚本
    ↓
检查 Excel 文件是否存在
    ↓
读取 sheet1（用户表）+ sheet2（广播表）
    ↓
连接 QQ 邮箱（IMAP）
    ↓
获取最近 N 封邮件的 UID 列表（默认 30 封）
    ↓
从最新邮件开始，逐封处理
    ↓
┌─────────────────────────────────────┐
│ 对每一封邮件：                        │
│  1. 检查是否已读 → 是则跳过            │
│  2. 提取发件人邮箱                     │
│  3. 检查发件人是否在 sheet1 中存在      │
│     → 不存在：标记已读，跳过            │
│     → 存在：继续                       │
│  4. 读取主题、正文                      │
│  5. 从正文提取 message_tag              │
│  6. 生成新的 message_id                │
│  7. 收集到待写入列表                    │
│  8. 将邮件标记为已读                    │
└─────────────────────────────────────┘
    ↓
所有邮件处理完毕
    ↓
一次性批量写入 sheet2
    ↓
打印运行总结，等待按键关闭
```

## ⚠️ 常见错误及解决方法

### 1. `❌ Excel 文件不存在`

**原因**：`EXCEL_FILE` 路径写错了。

**解决**：检查路径是否正确，使用绝对路径，注意 `r""` 前缀。

```python
EXCEL_FILE = r"D:\Dot2Dot\Version0.1\database1.xlsx"
```

---

### 2. `❌ sheet1 缺少 'user_id' 字段`

**原因**：Excel 的 sheet1 表头不正确。

**解决**：确保 sheet1 的第一行有 `user_id` 列，大小写必须一致。

| 正确 | 错误 |
|------|------|
| `user_id` | `UserId` / `user_ID` / 缺失 |

---

### 3. `❌ 邮箱连接失败`

**可能原因**：

| 原因 | 解决 |
|------|------|
| 授权码错误 | 重新生成 QQ 邮箱授权码 |
| 网络问题 | 检查是否能访问 `imap.qq.com` |
| 账号被限制 | 登录网页版邮箱确认状态 |

---

### 4. `❌ 写入 Excel 失败`

**原因**：Excel 文件被其他程序打开（如正在编辑中）。

**解决**：关闭 Excel 文件后再运行脚本。

---

### 5. `TypeError: can't compare offset-naive and offset-aware datetimes`

**原因**：日期比较时，一个带时区、一个不带。

**解决**：当前代码已统一使用 `timezone.utc`，不会再出现。

---

### 6. `cannot use a string pattern on a bytes-like object`

**原因**：邮件正文是 `bytes` 类型，但正则表达式期望字符串。

**解决**：当前代码中 `get_email_body()` 已加入类型判断，自动转换。
